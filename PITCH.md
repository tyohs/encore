# 🎤 みんカラ - ピッチ資料

## キャッチコピー
**「カラオケを、ライブに変える。」**

---

## 🎯 課題

### カラオケの「待ち時間」問題
- 歌わない人は**退屈**
- スマホをいじるだけの時間
- 歌っている人だけが盛り上がる
- リアクションが声だけで**伝わりにくい**

---

## 💡 解決策

### みんカラ = 全員参加型カラオケ

| 従来 | みんカラ |
|------|---------|
| 🎤 歌う人だけが主役 | 👥 全員でライブを作る |
| 😴 待つだけ | 🎸 バンドメンバーとしてゲーム参加 |
| 👏 声だけのリアクション | 📱 コール&リクエストがリアルタイム表示 |

---

## 🌟 主要機能

### 1. 3つの役割
```
🎤 シンガー：歌詞を見ながら歌う
🎸 バンド：リズムゲームで盛り上げる
🔦 オーディエンス：ペンライト＆コールで応援
```

### 2. リアルタイム連携
```
バンド/観客 → コール → 歌手画面にポップアップ！
バンド/観客 → リクエスト → 「手を振って！」が歌手に届く！
バンド/観客 → メッセージ → ニコニコ動画風に画面を流れる！
```

### 3. スコア消費型コール
```
リズムゲームでポイント獲得
    ↓
ポイントを使ってコール発動
    ↓
「いつ使うか？」の戦略性
```

### 4. 4方向スワイプ
```
    ⬆️ イェーイ！
⬅️ かわいい！   ➡️ サイコー！
    ⬇️ フゥー！
```

## 📝 作品概要（コピペ用）

```
【作品概要】
カラオケ参加者全員がスマートフォンで同一ルームに接続し、
それぞれの役割でリアルタイムに連携するWebアプリケーション。

＜基本機能＞
・ルーム作成/参加機能（ルームコードで共有）
・3つの役割選択（シンガー/バンド/オーディエンス）
・4曲の楽曲データ収録

＜シンガーモード＞
・歌詞のタイムライン表示
・盛り上がりゲージ表示
・バンドからのリクエスト受信・表示
・ファンサ完了ボタン

＜バンドモード＞
・4種の楽器選択（ドラム/ギター/キーボード/ベース）
・リズムゲーム形式のノート判定（PERFECT/GREAT/GOOD/MISS）
・スコア・コンボ表示
・スコア消費型コール機能（4種・4方向スワイプ対応）
・スコア消費型リクエスト機能（4種・30秒クールダウン）
・MP4アップロードによる背景映像再生
・振動フィードバック対応

＜オーディエンスモード＞
・ペンライト振り・色変更
・コールボタン（4種）
・ファンサリクエスト（3種）
・カスタムメッセージ送信

＜通信機能＞
・BroadcastChannel APIによるタブ間リアルタイム同期
・コール/リクエスト/メッセージの即時共有

＜技術構成＞
Next.js 16 / TypeScript / Tailwind CSS / Framer Motion / Zustand
```

## 📝 アピールポイント（コピペ用）

```
【アピールポイント】

1. 「待つ人」を「参加する人」に変える
   従来のカラオケでは歌わない人は退屈しがち。
   みんカラではバンドメンバーとしてリズムゲームに参加し、
   全員が一つのライブを作り上げる体験を提供する。

2. デジタルならではの双方向コミュニケーション
   声を出さなくても、スマホ操作でコールやリクエストを送信。
   歌手の画面にリアルタイムで表示され、反応が可視化される。
   「手を振って！」「ピース！」など具体的なアクションを依頼可能。

3. ゲーム性による能動的な参加
   コール/リクエストにはスコア消費が必要。
   リズムゲームでポイントを稼ぎ、ここぞという場面で使う
   戦略性が生まれ、最後まで集中して参加できる。

4. プロセカ風の没入体験
   ユーザーがMP4をアップロードすることで
   ノートの背景にMV映像を表示。
   音声と映像が自動同期し、ライブ感を演出。

5. 4方向スワイプで直感的操作
   上下左右のスワイプで異なるコールを発動。
   ボタンを探す必要なく、ジェスチャーだけで盛り上げられる。
```
